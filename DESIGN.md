# DESIGN.md — 攒攒

## 1. 设计结论

攒攒不是“AI 工具”，而是一个帮用户把真实意图说清楚、再连接到合适的人和机会的关系 Agent。界面首先要让人愿意说真话，其次才是展示系统能力。

视觉采用 **Warm Editorial Utility**：

- 从 SayBriefly 借鉴纸张感、强排版、友好而直接的语气和带印刷感的卡片；
- 产品工作台比营销页更安静，减少装饰和不必要的卡片；
- 以文字、编号、留白和状态变化表达智能，不用“机器人”“魔法”“发光”暗示 AI；
- 创业合作、招聘组队、恋爱交友共享同一套视觉语言，不按场景换主题皮肤。

关键词：`纸张`、`编辑排版`、`深森林绿`、`克制荧光黄`、`可信`、`有人味`、`不端着`。

## 2. 体验原则

### 2.1 先对话，再结构化

用户看到的主动作永远是“说说你现在想找什么”，不是填写表单。结构化信息由 Agent 从对话中逐步整理，并在关键时刻以可编辑卡片出现。

### 2.2 一屏一个重心

- 登录页：理解产品价值并进入；
- 对话页：继续表达；
- 匹配页：判断是否合适；
- 详情页：判断是否加入；
- 资料页：校正长期画像和当前意图。

任何页面只能有一个视觉最强的主按钮。

### 2.3 智能通过结果体现

智能感来自：记得上下文、少问重复问题、解释“为什么适合”、状态及时变化。不要用装饰性 AI 图标、动态光球或渐变制造智能感。

### 2.4 关系场景需要可信和克制

恋爱、招聘、创业都涉及真实的人。界面不应游戏化配对，不应使用“缘分值”“灵魂契合度”等夸张表达，也不应把用户变成一排头像商品。

## 3. 色彩系统

### 3.1 核心用色

| Token | 色值 | 用途 |
| --- | --- | --- |
| `canvas` | `#FDF9F2` | 全局纸张背景 |
| `canvasAlt` | `#F8F0DF` | 营销区块、侧栏、空状态 |
| `surface` | `#FFFDF8` | 卡片、对话输入区 |
| `ink` | `#1A3300` | 主文字、主按钮、强边框 |
| `inkSoft` | `#46523C` | 正文和辅助说明 |
| `inkMuted` | `#7B8175` | 元信息、占位文字 |
| `lineSoft` | `#D9D8CF` | 分隔线、默认边框 |
| `highlight` | `#FCEC5B` | 单一重点、主 CTA、匹配原因标记 |

### 3.2 使用比例

- 70% `canvas` / `surface`；
- 20% `ink` / 中性色文字和边框；
- 不超过 10% `highlight`；
- 荧光黄不作为大面积背景，不同时高亮三个以上区域；
- 深森林绿替代纯黑；禁止 `#000000` 作为品牌主色。

### 3.3 语义色

语义色只表达状态，不参与装饰：

- 成功 / 匹配中：浅苔绿色底 + 深绿色文字；
- 等待 / 待确认：浅麦黄色底 + 棕黄色文字；
- 危险 / 退出 / 删除：浅粉红底 + 暗红文字；
- 信息提示默认使用中性纸张色，不增加蓝紫色 AI 配色。

## 4. 字体与排版

### 4.1 字体角色

- 大标题：`Manrope` + 中文系统无衬线回退。圆润、紧凑、略带编辑标题感；
- 正文与界面：`Inter` + `Noto Sans SC` / `PingFang SC`；
- 标签与编号：`IBM Plex Mono` + 等宽中文回退；
- 不在同一页面引入第三套展示字体；
- 中文字体加载失败时必须保持字重和行高稳定，不允许布局跳动。

### 4.2 字号层级

| 层级 | 桌面 | 移动 | 字重 / 行高 | 用途 |
| --- | --- | --- | --- | --- |
| Display | `clamp(44px, 6vw, 84px)` | `40–52px` | 750 / 0.98 | 营销首屏一句话 |
| H1 | `40–52px` | `32–38px` | 720 / 1.08 | 页面主标题 |
| H2 | `28–34px` | `25–30px` | 700 / 1.2 | 区块标题 |
| H3 | `20–24px` | `19–22px` | 650 / 1.3 | 卡片标题 |
| Body L | `18px` | `17px` | 400 / 1.65 | 引导语、Agent 重点回复 |
| Body | `16px` | `16px` | 400 / 1.65 | 默认正文 |
| Small | `14px` | `14px` | 400 / 1.5 | 辅助说明 |
| Label | `11–12px` | `11–12px` | 500 / 1.4 | 状态、编号、元信息 |

### 4.3 排版规则

- 大标题每行尽量不超过 12 个汉字，使用语义换行，不按容器随机折行；
- 正文阅读宽度控制在 `42–68ch`；
- 中文正文不使用全大写式字距；英文小标签可使用大写和 `0.08em` 字距；
- 数字导航使用 `01`、`02`、`03`，不使用图标代替栏目名称；
- 营销文案短句化，不使用“重新定义、无缝、赋能、下一代、一站式”等 AI 营销套话。

## 5. 网格、空间与页面骨架

### 5.1 基础网格

- 4px 基础单位，常用间距为 8 / 12 / 16 / 24 / 32 / 48 / 64；
- 营销页最大内容宽度 `1320px`；
- 产品页最大内容宽度 `1440px`；
- 桌面侧栏 `260–288px`，内容区不得被侧栏覆盖；
- 对话主列最大宽度 `760px`，匹配结果可扩展到 `880px`；
- 卡片内部默认 `24–32px`，移动端 `18–20px`。

### 5.2 页面节奏

营销页允许大段留白与不对称构图；产品页采用稳定列宽和清晰分隔。不要把产品页做成连续的“大卡片墙”。相关内容优先通过：

1. 留白；
2. 细分隔线；
3. 标题层级；
4. 最后才是独立卡片。

## 6. 形状、描边与层次

- 默认卡片圆角 `20px`；小控件 `10–14px`；营销主视觉可到 `28px`；
- 主要卡片用 `1.5px` 深绿描边；次要卡片用 `1px` 柔和灰绿描边；
- 只在营销主卡、确认卡、重点推荐卡使用 `print` 错位阴影；
- 普通列表、表单和后台表格不使用错位阴影；
- 悬浮态最多上移 `2px`，禁止大幅漂浮；
- 不使用玻璃拟态、背景模糊、金属高光、霓虹外发光和多色渐变。

## 7. 品牌标识与图标

### 7.1 攒攒标识

- Agent 标识使用纯文字“攒”，置于简单方形或圆角方形容器；
- 产品 Logo 优先使用文字“攒攒”；
- favicon 使用单色 `Z` 或“攒”的抽象字形；
- 标识不加眼睛、嘴、天线或机器人特征。

### 7.2 图标规则

- 导航优先使用编号 + 文字：`01 和攒攒聊`；
- 必须使用功能图标时，采用同一套 1.5px 线性图标；
- 图标仅用于关闭、返回、展开、发送、搜索等通用操作；
- 场景入口使用文字标签：`合作`、`招聘`、`关系`；
- 禁止机器人脸、星芒、魔法棒、发光球、脑电路、火箭 emoji；
- 禁止 emoji 作为正式 UI 图标；
- 禁止为了填空而给每张卡片放一个图标。

## 8. 核心组件

### 8.1 按钮

**Primary**

- 深森林绿背景、纸白文字；
- 高度 `48–52px`，水平内边距 `20–24px`；
- 圆角 `14px`，字重 650；
- Hover：背景略亮并上移 1px；Active：回到原位；
- 同一视口通常只出现一个 Primary。

**Highlight CTA**

- 荧光黄背景、深绿文字和强描边；
- 只用于营销首屏注册、Intent 确认或加入局等关键转换；
- 不用于“取消”“返回”“编辑”类动作。

**Secondary / Text**

- Secondary 使用透明或纸白底、深绿描边；
- Text button 使用下划线或文字颜色变化，不伪装成无边界主按钮；
- 危险动作默认不做高饱和红色实心按钮，只有确认弹层内才强化。

所有按钮都必须具备 hover、active、focus-visible、disabled、loading 状态；触控热区不小于 `44×44px`。

### 8.2 输入框与对话输入区

- 输入框默认纸白底、柔和边框，Focus 切换为深绿 `1.5px`；
- 高度至少 `48px`；文本域自动增长但不超过 6 行；
- 错误信息紧邻字段，不用 Toast 代替表单错误；
- 对话输入区固定在页面布局底部，不遮挡最后一条消息；
- 发送按钮使用文字“发送”或简单箭头，不使用魔法棒；
- 空输入时发送按钮 disabled，Enter 发送，Shift+Enter 换行。

### 8.3 对话消息

- Agent 回复默认不做厚重聊天气泡，可使用左侧文字标识 + 自然段；
- 用户消息可使用浅米色或浅黄色小气泡，最大宽度 72%；
- 连续同角色消息减少重复头像和标签；
- 长回答使用短段落、清晰列表和必要的粗体，不生成彩色卡片堆；
- “正在理解”使用低调文字和三点节奏，不显示旋转机器人或发光动画；
- 消息失败时原位显示“重试”，保留用户输入。

### 8.4 Intent 确认卡

这是 Agent 从对话转向匹配的关键组件：

- 卡头使用等宽 `CURRENT INTENT / 当前意图`；
- 主标题一句话概括用户现在想找什么；
- 依次展示：目标、处境、可提供、投入方式、边界；
- 缺失项不展示空字段；
- “继续聊聊修改”为次按钮；
- “方向没问题，开始匹配”为 Highlight CTA；
- 确认后显示明确状态变化，不在历史中重复生成相同卡片。

### 8.5 推荐 / 局卡

- 每屏最多 2 个重点推荐，避免无限卡片瀑布流；
- 标题、局类型、人数、摘要形成主要层级；
- “为什么适合你”使用左侧 2px 荧光黄标记或局部高亮，不使用 AI 分数仪表盘；
- `查看详情` 为文字动作，`加入这个局` 为 Primary；
- 已加入状态显示 `已加入 · 退出`，两个动作必须可点击；
- 场景标签用短文字胶囊，但胶囊数量不超过 3 个；
- 成员不使用陌生人头像墙，默认以昵称首字、文字摘要和隐私状态呈现。

### 8.6 状态标签

- 使用 11–12px 等宽字体；
- 标签高度 `24–28px`；
- 颜色用于表达状态，不用于装饰；
- `匹配中`、`待确认`、`已暂停`、`已成局` 文案必须直接，不使用抽象图形让用户猜。

### 8.7 菜单

- 场景快捷入口放在明确命名的菜单内，不让孤立箭头承担功能；
- 左上入口点击后展示 `合作 / 招聘 / 关系 / 重新聊`；
- 左下账户菜单展示 `我的资料 / 管理后台（管理员）/ 退出登录`；
- 当前页面和当前状态必须可见；
- 点击外部、按 Esc、选择菜单项后关闭；
- 菜单项必须整行可点击，不能只有文字或小箭头可点。

### 8.8 表格与后台

- 后台提高信息密度，但继续使用纸张底和深绿文字；
- 表头 12px 等宽字体，行高不小于 52px；
- 操作放在行尾，危险操作需二次确认；
- 数据概览使用大数字 + 简短说明，不使用装饰性 3D 图表；
- 无数据时说明原因和下一步，不只显示空白插画。

## 9. 页面模板

### 9.1 登录 / 营销页

桌面采用 55/45 的不对称双栏：

- 左侧：一句大标题、短解释、三类真实场景；
- 右侧：登录 / 注册表单；
- 首屏文案明确“创业合作、招聘组队、恋爱交友都可以”；
- 不出现 Channel 选择；
- 场景案例使用三行编辑式文本，不做三个完全相同的功能卡；
- 可用局部荧光笔强调“先聊，再匹配”。

推荐首屏语气：

> 你想找的人，先说给攒攒听。

副文案直接解释：合作、招聘、加入团队或认真认识一个人，都从一段自然对话开始。

### 9.2 Agent 对话页

- 桌面左侧导航；中间为单一对话主列；右侧默认不常驻信息栏；
- 顶部只保留页面标题、Agent 状态和“重新聊”；
- 对话区独立滚动，页面整体不随消息异常跳动；
- 输入区在主列底部稳定可见；
- 推荐结果与 Intent 卡自然插入对话流，但最新一组结果替换旧结果，避免重复堆叠。

### 9.3 发现新局

- 顶部先给一句编辑式标题和总数，再放筛选；
- 桌面最多两列，信息较长时使用单列；
- 不同场景共享布局，只通过文字标签区分；
- 筛选条可横向滚动但隐藏滚动条，当前项有清晰深绿底或下划线。

### 9.4 局详情

- 左侧为局信息和 Agent 群体观察；右侧为加入状态与成员摘要；
- “问攒攒是否适合”是可继续追问的次级动作，不生成一次性假评分；
- 关系场景隐藏不必要的个人敏感信息；
- 加入、退出、已满、已截止状态都必须有明确反馈。

### 9.5 我的资料与意图

- 桌面采用资料 / 当前意图双栏，但视觉主次明确；
- “编辑资料”进入真实表单或抽屉，不能是无响应按钮；
- 长期资料和当前意图区分标题和状态；
- 暂停匹配后立即更新状态、按钮文案和侧栏提示。

## 10. 动效与反馈

- 页面进入：`opacity 0→1` + `translateY 8px→0`，220–360ms；
- 卡片 Hover：最多 `translateY(-2px)`；
- 菜单：140ms 淡入和轻微位移；
- 新消息：轻微淡入，不逐字打印整段长文；
- Skeleton 只用于确实需要等待的内容，颜色接近纸张，不用闪烁高光；
- 支持 `prefers-reduced-motion`，减少位移和循环动画；
- 所有异步动作必须有 loading、success 或 error 反馈，禁止点击后没有反应。

## 11. 响应式

### Breakpoints

- `sm`: 480px；
- `md`: 768px；
- `lg`: 1024px；
- `xl`: 1280px。

### 移动端规则

- 桌面左侧栏改为底部导航或顶部简化菜单；
- 对话主列占满宽度，左右内边距 16px；
- 输入区考虑 `env(safe-area-inset-bottom)`；
- 双栏页面在 1024px 以下改为单列；
- Primary 按钮在关键确认场景可全宽；
- 弹层优先使用底部 Sheet，保留明确关闭动作；
- 不使用固定 `100vh`，使用 `100dvh`；
- 任何页面不得横向溢出。

## 12. 可访问性

- 正文与背景对比度达到 WCAG AA；
- Focus-visible 使用 2px 深绿外框 + 2px 纸张间隔；
- 不能只靠颜色表达匹配、暂停或错误状态；
- 表单字段有可见 label；
- 图标按钮必须有 `aria-label`；
- 键盘可以完成登录、菜单、对话输入、确认意图、查看详情和加入局；
- Toast 不承载唯一关键信息；
- 动态 Agent 消息用合适的 live region，避免一次朗读整个历史对话。

## 13. 文案语气

攒攒说话像敏锐、尊重边界的熟人：简短、具体、不评判。

**应该：**

- “你更想先找一位一起做产品的人，还是先验证方向？”
- “我理解你能投入周末时间，但暂时不考虑全职，对吗？”
- “这个局适合你，因为双方都想先做一个两周的小实验。”

**不要：**

- “让我用 AI 魔法帮你精准匹配灵魂伙伴。”
- “基于大模型的多维度智能分析，我们为你生成了 98% 契合度。”
- “开启你的无限可能。”

每次最多追问一个关键缺口。按钮文案使用动作结果，例如“开始匹配”，而不是模糊的“确定”。

## 14. 禁止项

- 禁止机器人、星芒、魔法棒、发光球、脑电路和火箭作为 AI 符号；
- 禁止紫蓝渐变、玻璃拟态、霓虹光和粒子背景；
- 禁止 emoji 作为正式产品图标；
- 禁止每个区块都是同尺寸圆角卡；
- 禁止三列等宽“功能卡 + 图标 + 两行文案”的 AI 模板结构；
- 禁止头像墙、虚构用户照片和未经授权的人像；
- 禁止无意义的匹配百分比、雷达图和能力评分；
- 禁止把创业和恋爱拆成视觉上互斥的两个产品；
- 禁止固定问卷式 Step 1 / Step 2 / Step 3 主流程；
- 禁止只有箭头或省略号可点击却没有菜单反馈；
- 禁止按钮点击无 loading、无状态变化、无错误说明；
- 禁止复制 SayBriefly 的 Logo、插画、角色形象、截图或品牌专有文案。

## 16. Agent / 编码执行指令

在生成或修改任何界面前：

1. 先判断页面唯一主任务；
2. 使用本文件的颜色、字体、圆角和间距 Token，不临时发明新视觉语言；
3. 优先用排版和留白建立层级，再决定是否需要卡片；
4. 把 Agent 表现为会理解上下文的文字角色，不做机器人视觉化；
5. 所有交互必须实现完整状态，不能只画静态按钮；
6. 创业、招聘、恋爱场景共享组件和主导航；
7. 若设计与可用性冲突，优先可读、可点击和状态明确；
8. 完成后按“禁止项”逐条自检。

## 17. 验收清单

- [ ] 首屏说明创业合作、招聘组队、恋爱交友都支持；
- [ ] 没有 Channel 二选一；
- [ ] 对话体验不是固定问卷；
- [ ] 全站没有机器人脸、星芒、魔法棒和 emoji 产品图标；
- [ ] 页面只使用纸张色、深森林绿和克制荧光黄作为核心品牌色；
- [ ] 同一视口只有一个最强主动作；
- [ ] Intent 卡可修改、可确认；
- [ ] 推荐卡解释为什么适合，但不使用虚假百分比；
- [ ] 菜单整行可点，并支持 Esc / 点击外部关闭；
- [ ] 编辑、加入、退出、暂停、重新聊等动作有真实反馈；
- [ ] 桌面和移动端无横向溢出；
- [ ] 键盘焦点、对比度和触控热区合格；
- [ ] 没有复制参考站品牌资产或专有文案。
